MySQL 索引类型及分类

一、从数据结构角度分类

1. B+树索引（最常用）

  • 特点：

    • MySQL默认的索引类型（InnoDB使用B+树）

    • 所有数据按顺序存储在叶子节点

    • 叶子节点之间有指针形成链表，便于范围查询

    • 高度平衡，查询性能稳定

  • 适用场景：

    • 等值查询（=）

    • 范围查询（>、<、BETWEEN、LIKE ‘abc%’）

    • 排序（ORDER BY）

    • 分组（GROUP BY）

2. 哈希索引

  • 特点：

    • 使用哈希表存储数据

    • 只支持等值查询（=）

    • 查询速度极快（O(1)时间复杂度）

    • 不支持范围查询、排序和分组

    • InnoDB有自适应哈希索引（AHI），但不可手动创建

  • 适用场景：

    • 内存表（MEMORY引擎）

    • 等值查询非常频繁的场景

3. 全文索引（FULLTEXT）

  • 特点：

    • 专门用于文本内容的搜索

    • 支持自然语言搜索和布尔搜索

    • InnoDB和MyISAM都支持（InnoDB从5.6开始支持）

  • 适用场景：

    • 大量文本搜索（如文章内容）

    • 关键词匹配

4. R树索引（空间索引）

  • 特点：

    • 用于地理空间数据

    • MySQL通过GIS扩展支持

  • 适用场景：

    • 地理位置查询（如经纬度范围）

二、从物理存储角度分类

1. 聚簇索引（Clustered Index）

  • 特点：

    • 数据和索引存储在一起

    • InnoDB中主键就是聚簇索引

    • 每个表只能有一个聚簇索引

    • 叶子节点存储整行数据

  • 优点：

    • 范围查询效率高

    • 减少IO次数（数据就在索引中）

  • 缺点：

    • 插入速度可能较慢（需要维护顺序）

2. 二级索引（Secondary Index）

  • 特点：

    • 独立于数据存储的索引

    • InnoDB中二级索引存储的是主键值

    • 查询时可能需要回表（通过主键再查一次聚簇索引）

  • 适用场景：

    • 非主键字段的快速查找

  • 特点：

    • 索引列的值必须唯一

    • 可以有一个NULL值（除非同时指定PRIMARY KEY）

4. 主键索引

  • 特点：

    • 特殊的唯一索引

    • 不允许NULL值

    • InnoDB中自动成为聚簇索引

  • 示例：

5. 全文索引

  • 如前所述，专门用于文本搜索

四、从功能角度分类

1. 普通索引

  • 最基本的索引类型

  • 只加速查询

2. 唯一索引

  • 加速查询 + 保证数据唯一性

3. 主键索引

  • 加速查询 + 唯一约束 + 表标识

4. 组合索引

  • 多列组合的索引

  • 需要考虑最左前缀原则
